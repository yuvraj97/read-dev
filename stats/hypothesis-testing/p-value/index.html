In Type \(1\) Error we are specifying a level (say \(\alpha\) ) and then testing that "can we reject the test at level \(\alpha\)".
If we can reject the test at a level say \(5\%\) what about rejecting it at a level \(1\%\) or \(2\%\) or \(10\%\) ... .
If we have a some notion for minimum level at which we can reject the null hypothesis(assuming that null hypothesis is true), then we can answer for all of those questions.

What we are currently doing is we are specifying a level \(\alpha\) and based on that level we are finding a cut-off for our test statistics beyond which we can reject the null hypothesis.
And here we want to find minimum level at which we can reject the null hypothesis, so we are finding a cut-off for our level \(\alpha\).
And this minimum level is known as \(\text{p-value}\).
So.

Asymptotic \(\text{ p-value }\) of a test \(\psi_\alpha\) is the smallest asymptotic level \(\alpha\) at which our test \(\psi_\alpha\) rejects the null hypothesis \(H_0\).
\(\text{ p-value }\) is the smallest level at which we can reject the null hypothesis.
So if we go a little above the \(\text{ p-value }\) we reject the null hypothesis, and below it we fail to reject the null hypothesis.

\(\text{ p-value }\) is not Type 1 Error, it will be more clear when we talk about "What if \(\text{ p-value }\) \(\to 0 \) " below.
The smaller the \(\text{ p-value }\), more confidently we can reject the null hypothesis \(H_0\)

What if \(\text{ p-value }\) \(\to 0\)

For Type \(1\) Error,   we decide our level \(\alpha\)
If we decide level \(\alpha \to 0\) then we reject less and less.
If we decide that accepted maximum Type \(1\) Error is \(0\) then we never reject the null hypothesis, remember our Criminal Trial example where we discuss what does Type \(1\) Error \(=0\) looks like.
So if we make level extremely small then we have to go with that dumb test which always fails to reject.

But here we are saying that "The smaller the \(\text{ p-value }\), more confidently we can reject the null hypothesis \(H_0\)"
So we don't have any problem for an extremely small level(\(\text{ p-value }\)), even we are saying it's good to be as small, buy why?

Because in Type 1 error we decides what our level \(\alpha\) will be and based on our decision we make a test \(\psi_\alpha\).
So for an (asymptotically) Normally distributed test statistics \(T_n\), if we decide that maximum accepted Type \(1\) Error is \(0\) So \(\alpha=0\) (this is our decision), then we have no choice but to choose \(\infty,-\infty\) as boundaries for our test \(\psi_\alpha\).
So we have no choice but to, never reject the null \(H_0\).

But in \(\text{ p-value }\) we don't decide any level a priory.
Our sample(data \(X_1,\cdots,X_n\)) itself decides a level (which we call \(\text{ p-value }\)).

Recall our Coin Toss Example
Here we reject if there is a discrepancy between \(p\) and \(\hat{p}_n\) (estimator of \(p\))
(remember here we assume that the null hypothesis is true.)
Say that our test statistics is \(T_n = \sqrt{n}\frac{\hat{p}_n-p}{\sqrt{p(1-p)}}\)
(It was not the same test statistics as in our previous Coin Toss Example , but I needed this test statistics for more clear explanation, I changed the test \(\psi\) accordingly).
\(T_n\) is (asymptotically) Normally distributed test statistics.
In our coin toss example according to our data \(X_1,\cdots,X_n\) when will we reject the null Hypothesis?
We reject the null hypothesis if our test statistics \(|T_n|\) is greater then something.
\(\psi = 𝟙\{|T_n|\gt C\};\quad C\in\mathbb{R}^+\)
We have a realization of a test statistics say \(T_n'=T_n(X_1\cdots X_n)\)
According to our data, what is the minimum level (\(\text{ p-value }\)) at which we reject the null hypothesis?
We reject the null if any test statistics \(|T_n|\) if greater then \(|T_n'|\) .
(remember according to our assumption null is true, so this is an error).
so \[\text{ p-value }=\mathbb{P}_{\theta\in\Theta_0}\left( |T_n| \gt |T_n'| \right)\] and because \(T_n\) is (asymptotically) Normally distributed.
\[\text{ p-value }=\mathbb{P}_{\theta\in\Theta_0}\left( |Z| \gt |T_n'| \right);\quad Z\sim \mathcal{N}(0,1)\]
Now we can see that \(\text{ p-value }\) is decided by our sample.
So a very small \(\text{ p-value }\) means that our test stastics \(|T_n|\) is very large, which is very unlikely to happen "by a chance" so very small \(\text{ p-value }\) gives us a very strong evidence against the null hypothesis.

After we get our \(\text{ p-value }\), we just need a threshold for the \(\text{ p-value }\), beyond which we reject the null hypothesis.
For our Coin Toss Example say we choose a threshold of \(5\%\).
So it we get \(\text{ p-value }\) below \(0.05\) we reject the null hypothesis, otherwise we fail to reject the null hypothesis.

If you are curious why is there a little inconsistency in Test statistics
Here we take \(T_n = \sqrt{n}\frac{\hat{p}_n-p}{\sqrt{p(1-p)}}\) where as in Coin Toss Example we took \(T_n = \sqrt{n}\frac{|\hat{p}_n-p|}{\sqrt{p(1-p)}}\)
We should have taken \(T_n = \sqrt{n}\frac{|\hat{p}_n-p|}{\sqrt{p(1-p)}}\)
But I don't want to introduce Folded Normal Distribution
\(|Z|\sim\)Folded Normal Distribution, where \(Z\sim\mathcal{N}(\mu,\sigma^2)\)
